Html Guide

This is my Html guide. It includes all the Html code I used to create the main page of this site. You can find my CSS and Java Script here.

The Boilerplate

I started with the HTML document structure by opening a new file in Visual Studio Code (which is a text editor or an Integrated Development Environment (IDE)) and just by typing an explanation mark at the top of the page, tells the app to create a Html boiler plate which automatically creates the basic structure of an HTML document for you.

This creates a boilerplate, or Html document structure:

  1. <!DOCTYPE html> This declaration at the beginning of the document specifies that it is an HTML5 document.

  2. < html> This element is the root element of an HTML page.

  3. <head> This contains meta-information about the HTML document, such as the title, character encoding, CSS stylesheets, JavaScript files, and other metadata.

  4. <meta charset="UTF-8"> This meta tag specifies the character encoding for the document, ensuring that it can handle a wide range of characters.

  5. <title> This sets the title of the web page, which is displayed in the browser's title bar or tab.

  6. <body> This represents the main content of the web page and contains all visible content, such as headings, paragraphs, images, links, etc.

  7. <h1> is a top-level heading on the page and so also usually the largest.

  8. <p> This element represents a paragraph of text.


You can customize this boilerplate by adding your own content, styles, and scripts to create your web page. It serves as a foundation that ensures your HTML document has the necessary structure and elements to function correctly and be properly displayed in web browsers.

The Head

So now with my boilerplate ready I added some external reources to the head of the document:

1. Inside the <head> tags, I added the <link> tag to link the page to my external CSS stylesheets. The href attribute specifies the path to the CSS file, in this case, "/styles.css".

2. Also within the <head> tags, I added another <link> tag to set the favicon, which is the small icon displayed in the browser tab. The href attribute specifies the path to the favicon file, in this case, "./images/favicon.ico".

3. Still within the <head> tags, and again adding the <link> tags, I was able to preconnect to external resources and import Google fonts. This helps improve the performance of the webpage by initiating an early connection to the specified domains. In this case to be able to display my chosen font correctly to all users. 8. The href attribute specifies the URL of the font stylesheets to be imported.

The Body

After the closing </head> tag, open the <body> tag to start the body section of the webpage.

The content of the body of the site is made up of five divs with individual classes. The class "top-container" contains the content for the top section of the webpage, the class 'middle-container' for the middle section, the div holding the classes 'skills' and 'skill-row2'for the third section, class 'contact-me' for the fourth and finally the class 'bottom-container' for the footer.

More about divs

HTML div tags, (short for "division,") are used to create a container or a division within an HTML document. They are a fundamental part of HTML and are used to group and organize content, providing structure and layout to web pages. The primary purpose of the <div> tag is to divide or section off parts of a web page, allowing you to apply specific styles, formatting, or functionality to that particular section. By using div tags, you can group related elements together and apply CSS (Cascading Style Sheets) styles to them collectively. Here are some common use cases and benefits of using <div> tags:

  • Structural Organization: <div> tags are used to create a logical structure for your web page, dividing it into smaller, manageable sections. For example, you can use div tags to separate the header, navigation, content area, sidebar, and footer of your page.

  • CSS Styling: Div tags provide a way to target specific sections of your HTML document and apply CSS styles to them. By assigning class or ID attributes to div tags, you can selectively style them using CSS rules, such as changing background colors, fonts, margins, or applying borders.

  • Layout and Grid Systems: Div tags are commonly used to create flexible layouts and grid systems. By combining div tags with CSS properties like display: flex or using CSS frameworks like Bootstrap or Foundation, you can create responsive designs that adapt to different screen sizes and devices.

  • Content Grouping: Div tags allow you to group related elements together, such as images, paragraphs, headings, lists, or forms. This grouping makes it easier to apply consistent styles or manipulate the content as a whole using JavaScript or other scripting languages.

  • Scripting and Interactivity: Div tags can serve as containers for interactive elements, such as buttons, dropdown menus, tabs, or accordions. You can attach event handlers or apply JavaScript libraries to the divs to enable interactivity and dynamic behavior on your web page.

To use a div tag, you simply enclose the desired content within opening <div>and closing </div> tags. Overall, <div>tags are versatile and essential elements in HTML that provide structure, layout, and flexibility to web pages, allowing for easier management and presentation of content.
